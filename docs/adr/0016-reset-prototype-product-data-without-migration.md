# Reset prototype product data instead of migrating it

The browser-first versioned model starts without migrating existing prototype documents. The controlled reset deletes all OctoMeta document content, normalized graph/block/chip/undo/workbook rows, versions, access and migration state, and document assets, while preserving Better Auth account data required for existing users and preserving the separate marketing waitlist; destructive execution requires verification of the exact deployment and table allowlist.
