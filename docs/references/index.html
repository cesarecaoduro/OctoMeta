§ 00THE PROBLEM
Why OctoMeta exists

The calc you stamped isn't the calc you have.

Every engineer knows the Monday: the span changed on Friday, the spreadsheet was updated, the Word report wasn't, and the model belongs to another team. Three artifacts. Three different truths. One signature — yours.

=C4*SUM(D6:D12)/B33

Logic you can't review.

That formula is load distribution across a six-pile group — but nothing on screen says so. Cell references hide engineering intent, so checking means reverse-engineering. And reverse-engineering means, on most projects, it doesn't happen.

report_FINAL_r3.pdf · issued 08 Nov
model updated 12 Nov

Deliverables that rot.

The PDF was true the day it was printed. Four days later the founding level changed and the report didn't. Now the stale copy carries your stamp while the truth lives in somebody else's file — and nobody is sure which numbers were checked.

234.6 kN + 234.6 kip = ?

Units on the honour system.

Spreadsheets will add anything to anything. A kip that should have been a kilonewton survives every review that doesn't recompute by hand — the exact class of silent error behind some of the industry's most expensive failures.

~40%
of engineering time goes to finding, rebuilding and reconciling calculations
~24%
of computational notebooks re-execute without error — order and hidden state defeat the rest
up to 7%
of contract value on large projects attributed to calculation errors (industry estimates)

The root cause is the same in every case: the calculation, the narrative and the geometry are copies of each other, maintained by hand. OctoMeta removes the copies. One graph underneath — so there is nothing to reconcile, nothing to go stale, and nothing to take on faith.

§ 01ONE GRAPH
Reactive by construction

Runs in any order — the graph decides.

Position is for humans. Order is for the graph. OctoMeta evaluates by dependency, not by where a block sits — your document reads top-to-bottom while it computes edge-by-edge, like an octopus moving eight arms with one mind.

§ 02UNITS
Unit-safe by default

Every number carries its units.

Dimensional mismatches surface as errors, not silent bugs. Show the full substituted derivation on demand — so a checker reads the logic, not cell references.

q_app=P / B²
=800 kN / (2.0 m)²=200.0 kPa
P + B→#UNIT! kN + m

"Show steps" renders the derivation with values and units substituted. Mixing dimensions is a typed error that propagates — never a silent wrong number.

§ 06CONNECTED
Your tools, one graph

Connected to the AECO tools you already run.

The numbers you need already exist — in Revit models on Autodesk Construction Cloud, and in the business systems that hold your rates and quantities. OctoMeta binds them into the graph instead of asking you to retype them.

Autodesk Platform Services now publishes official MCP server samples for its AEC Data Model API — hubs, projects, element groups and category-filtered elements from Revit models, queried over GraphQL, no plugins. OctoMeta consumes exactly this class of server: bind an element property as a named value and it enters the graph with provenance, like any other input. The same pattern reaches your business systems — point OctoMeta at the MCP server in front of your ERP, and steel rates land in the same checked graph as steel tonnage.

First connectors ship read-only: Revit via APS, plus any MCP server you already run. Exposing the document graph as its own MCP server is on the roadmap, built on the same mutation API.
