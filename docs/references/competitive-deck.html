§ 01 · Why now

The calc you stamped isn't the calc you have.

  • =C4*SUM(D6:D12)/B33 — engineering intent hidden behind cell references. Checking means reverse-engineering, so it mostly doesn't happen.
  • report_FINAL_r3.pdf · issued 08 Nov vs model updated 12 Nov — the deliverable rots the moment it's printed, and the stale copy carries the signature.
  • 234.6 kN + 234.6 kip — units on the honour system. Silent, and behind some of the industry's costliest failures.

~40% of engineering time reconciling calcs · ~24% of notebooks re-execute cleanly · up to 7% of contract value lost to calc errors. Root cause: three hand-maintained copies of one truth.

Cloud calc pages (MathJS + Python), templates, collaboration, AI generate-and-verify. A$3.8M pre-seed; users at Arup, Jacobs, Aurecon, WSP; $24–34 /user/mo.

READ
  • Respect: collaboration, enterprise traction, human-verified AI philosophy.
  • Gap we attack: parameters on pages, not a typed graph. No geometry, no live 3D, no IFC — their answer to geometry is "round-trip through Grasshopper."
§ 05 · The units incumbents

Blockpad & Mathcad Prime

  • Blockpad (~$18/mo): best-in-class units UX, readable calcs, 2D sketches. Ceiling: no 3D, no IFC, no AI, desktop-first.
  • Mathcad Prime 12 (~$735–860/yr): the trust anchor. Ceiling: desktop, price, no geometry-from-calc, weak collaboration.
  • Table stakes they set for us: unit parity, show-steps, readable derivations. We must match before we differentiate.

§ 07 · Live threats

Two fronts to watch.

  • Excel + Copilot (~$21/user/mo add-on): AI momentum on the tool everyone owns. But Copilot writes into opaque cells — it amplifies the reviewability problem we solve. Their anchor sets buyer psychology.
  • Open Calculations Studio (OpenAEC, ex-CalcPAD, v0.1.x, free/LGPL): already ships live IFC4x3 export from calcs. Desktop, no graph, no 3D viewer, no grid — but it proves our headline thesis in the open. Trigger: if they add a dependency graph + viewer, we accelerate.

§ 09 · Connected

Model data and cost data, in the same checked graph.

  • What exists today: Autodesk Platform Services publishes official MCP server samples for its AEC Data Model API — GetHubs · GetProjects · GetElementGroupsByProject · GetElementsByElementGroup — Revit data on ACC over GraphQL, with reference implementations in .NET, Python and JavaScript and published best-practice guidance.
  • Example 1 — model data: bind footing.width ← Tower-B.rvt · footing F3 · width through an APS MCP server. It enters the graph as a typed, provenance-stamped input; re-pull and the whole document recomputes.
  • Example 2 — cost data: rates.concrete ← erp · unit_rate("N40 mix") via the MCP server in front of your ERP — so cost = VOLUME(footing) × rates.concrete lives beside the engineering, unit-checked and reviewable.
  • Scope honesty: first connectors are read-only (APS + any MCP server you run). Exposing the document graph as an MCP server is roadmap, on the same mutation API.

POSITIONING LINE → "Connect your AECO tools and ERPs. Stop retyping your own model."
